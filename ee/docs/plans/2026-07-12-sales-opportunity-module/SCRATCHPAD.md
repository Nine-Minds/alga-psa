## Decisions

- (2026-07-12) **Build the module at all**: passes the product criterion ("teach an MSP owner to grow their business") only if built as an MSP growth engine, not a generic CRM. Generic contacts/deals/stages CRM explicitly rejected — HubSpot free tier already wins that game. Differentiator: AlgaPSA owns the downstream objects (quotes, agreements, projects, billing history) that deals come from and turn into.
- (2026-07-12) **Design thesis — three jobs**: (1) *Find the money* — pipeline generators computed from owned data (renewals at T-120, T&M-to-agreement candidates from billing, whitespace grid, asset aging); (2) *Force the follow-through* — one next-action-with-date per open opportunity, staleness nudge/escalation ladder; (3) *Own what the deal becomes* — opportunity → quote → agreement + onboarding project spine.
- (2026-07-12) **No separate leads entity.** Prospects are lightweight `clients` rows with a new `lifecycle_status` (prospect/active/former). Rationale: the lead/account/deal taxonomy is exactly the ceremony that killed HubSpot/Salesforce adoption for both research personas; a prospect-as-client gets tickets/quotes/interactions for free. The workflow event catalog already names `CLIENT_STATUS_CHANGED` "prospect→active" — vocabulary existed, schema didn't.
- (2026-07-12) **Opinionated evidence-derived stages; no configurable pipelines, no stage percentages.** Stage floor derives from verifiable facts the system observes (quote sent_at, assessment sold, decision-maker meeting scheduled, contract sent); rep declares confidence separately; the gap feeds calibration reporting (EE). Sam interview: stage-percentages "multiply two lies."
- (2026-07-12) **Work queue is the primary surface** (next actions due + fired generator triggers), not a kanban funnel. Dana interview: empty/stale funnel board = guilt mirror → abandonment. Board exists as a secondary view.
- (2026-07-12) **MRR/NRR are first-class split values** on opportunities (plus hardware flagged separately). Single blended deal value rejected (Sam: "$200k at $3k MRR is a different animal from $150k of hardware").
- (2026-07-12) **Automation boundary: draft, never send.** Auto-draft follow-ups, auto-create renewal opportunities, auto-log, auto-downgrade stage with stated reason = allowed. Auto-sending anything to a client = never (unanimous across personas).
- (2026-07-12) **CE/EE seam** (user-confirmed): CE = solo-owner growth engine — opportunity object, work queue, next-action discipline, staleness ladder (solo mode = self-interrupt), generators against own data (incl. whitespace grid), evidence-derived stages, MRR/NRR, full quote/agreement/project spine, basic pipeline reports. EE = the management layer — per-seller calibration, forecast floor/ceiling bands, pipeline meeting mode, commitments ledger with enforced close gates, QBR program (trigger packs assigned to AMs), multi-office rollups, custom methodology workflows via the (already-EE) Designer. **All AI features (drafting, email mining/auto-logging) gated behind the existing AI module seam** (user-confirmed 2026-07-12). Peer benchmarks = cloud/EE, later plan.
- (2026-07-12) **Methodology ships as default-on system workflows** where the workflow engine fits (nudges, escalation, renewal generation), so EE can customize via Designer while CE gets working defaults.
- (2026-07-12) **Design language locked (user-confirmed): the "Docket + Advisor" hybrid** — canonical mockup `mockups/work-queue-hybrid.html`, rules codified in PRD UX section: finishable column with a bottom; evidence-assembled why-sentences (template-composed, true without AI — AI upgrades fluency, not honesty); one primary button per screen; dollar-forward found-money cards; lesson strip teaching with the tenant's own numbers; greeting carries the found-money total. Added F114 (why-sentence composer) + F115 (lesson strip) + T044.
- (2026-07-12) **Implementation logistics (user instruction)**: Claude does ALL UI work personally (owns the design); everything else (migrations, models, actions, API, events, generators, reports backend) is offloaded to codex with model gpt-5.6-sol at xhigh reasoning effort; house voice/copy per the alga-tech-doc-writing skill.
- (2026-07-12) **User-confirmed scope calls**: plan covers CE + EE management layer + light AI tie-ins (follow-up draft generation and a per-user "write this in my voice" steering profile — behind the AI module seam; mailbox mining/auto-logging still out). Name = **Opportunities** (package `packages/opportunities`, resource `opportunities`, nav `/msp/opportunities`). Stage model = 6-stage assessment-centric (Identified → Qualified → Assessment → Proposed → Verbal → Won/Lost), evidence-derived with skippable checkpoints. Simple kanban board ships in CE v1 alongside the queue (meeting mode stays EE).

## Discoveries / Constraints

- (2026-07-12) `quotes.opportunity_id` already exists — nullable, FK-less placeholder added by quoting Phase 6 explicitly ahead of a CRM module (`ee/docs/plans/2026-03-13-quoting-system/SCRATCHPAD.md:288`). Carried through create/edit/convert paths and REST schemas already.
- (2026-07-12) No opportunity/lead/pipeline/deal tables exist anywhere (823 server migrations + ee migrations checked). Greenfield is the opportunity data model + methodology layer only.
- (2026-07-12) `client_type` is only `'company' | 'individual'`; the sole status field is boolean `is_inactive`. Adding `lifecycle_status` is a real schema change with UI/filter implications across client lists.
- (2026-07-12) Workflow engine already ships a CRM event category (`CLIENT_STATUS_CHANGED`, `INTERACTION_LOGGED`, …) and a `crm.ts` business-operations module; EE plans `2026-04-25-workflow-crm-actions/` + `-followup-actions/` roadmap `crm.create_quote`, `crm.schedule_activity`, etc. Only `crm.create_activity_note` live today. Many catalog entries are "proposedEvents" — check emission per event before relying on one.
- (2026-07-12) Renewal machinery half-exists: `client_contracts` renewal fields + renewal work-item queue (`IClientContractRenewalWorkItem`, statuses pending/renewing/non_renewing/snoozed/completed). Renewal generator must integrate with (not duplicate) this queue.
- (2026-07-12) `interactions` is a working activity/timeline substrate (types, statuses, ticket/project links); reuse for opportunity timeline instead of a new activity table. Tags enum (`TaggedEntityType`) has no `quote`/`opportunity` — needs extension.
